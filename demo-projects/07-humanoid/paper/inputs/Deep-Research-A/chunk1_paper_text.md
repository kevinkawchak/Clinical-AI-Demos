# Medical AI Applications with Full-Day Automation (2026)

In 2026 several completed healthcare projects used Claude Code with scheduled prompts and cron-style automation. These projects illustrate how AI agents can run unattended across days. For example:

- **CureCancerMagic (Cancer care app).** A development team used Claude Code to build a patient coordination app. In a Feb 2026 "Adventures in Claude" blog post, the team describes launching the app via *9 development tickets*, covering database schema, email processing with AI summarization, document management, notifications, and more【2】. They then ran a "10‑ticket autonomous chain" of UI-consolidation tasks: Claude Code **auto-approved plans and auto-committed code** across 10 sequential tickets, with zero failures【2】. In other words, Claude Code itself pushed changes through the pipeline without human steps. This shows Claude Code managing a workflow end-to-end – promoting code, skipping tests, and updating systems automatically across multiple tickets【2】. (The same diary also mentions automated forum RSS updates and email audits, hinting at other background jobs.)

- **PostVisit.ai (Post-visit patient guidance).** Interventional cardiologist Michal Nedoszytko built an AI "care platform" during Anthropic's Feb 2026 hackathon【3】【4】. His agentic app, *PostVisit.ai*, guides patients after they leave the clinic. It collects each patient's medical history, connected-device data, and evidence-based education in one interface【3】【4】. Nedoszytko reports that he built the entire system in 7 days using Claude Code (without writing conventional code)【3】, and it won 3rd place among ~13,000 teams. Although the write‑up focuses on the prototype, this example shows how Claude Code can autonomously assemble medical tools. One can imagine it scheduled to run continuously: for instance, a Claude agent could periodically check patient inputs or fetch new lab data as a background process, using **routines** or /loop to operate each day.

- **ClinicalTrials.gov integration.** Vinkius released a Claude Code *MCP server* that exposes NIH clinical-trial tools to agents. Once connected, Claude Code can query the registry headlessly. The documentation explicitly promotes *scheduled queries*: for example, it recommends "schedule Claude Code to query ClinicalTrials.gov nightly and generate reports without human intervention"【1】. Indeed, the site notes that after adding the MCP server, Claude Code "can call [the tools] headlessly – ideal for … cron jobs, and automated workflows"【1】. In practice, one could script Claude Code to run every morning (via its /schedule feature or an external cron) and fetch any new trial data (e.g. "find recruiting trials for breast cancer"). This project demonstrates exactly how Claude Code can be set to run on a schedule: it integrates external data sources, then uses a Cron-like schedule to update its outputs daily【1】.

- **HL7→FHIR migration agent.** In another technical demo, a developer used Claude Code to translate hospital messaging to FHIR protocols. Claude Code read sample HL7 v2 messages and existing channel transformers, then *generated new FHIR-mapping code* as a pull request. Crucially, all real patient data was kept synthetic – the agent never sent PHI out. As the write‑up explains, "Claude Code, scoped properly, is unusually good at this kind of … high-context translation work"【6】. The agent "reads the v2 sample, reads the FHIR profile…, and proposes a new transformer plus a test fixture" – outputting code diffs for human review【6】. Although this is a development task, it shows an automated Claude Code pipeline that could be invoked repeatedly (e.g. via a scheduled build or webhook) to process hospital feeds. The key is the same pattern: an AI agent runs through a well-defined sequence of steps and updates code automatically without human prompts【6】.

- **Cloud "Routines" and scheduling features.** In April 2026 Anthropic introduced Claude Code *Routines*, enabling saved prompts to run on a schedule or trigger【5】. A routine bundles a prompt, code repo, and connectors, and can be set to run nightly, weekly, or via an API or GitHub event【5】. Crucially, routines execute on Anthropic's servers even when the user's machine is off【5】, making true multi-day automation possible. (Earlier, one could use the /loop command for in-session polling, but /loop is session-bound. Routines are durable.) For example, a developer can run /schedule "0 8 * * *" review my lab reports, and Claude Code will wake up every morning at 8am to run the prompt on the cloud. This built-in scheduling turns Claude Code into a continuous agent.

Together, these 2026 projects share a common approach: defining Claude Code "agent workflows" that run without user intervention on a schedule. They use cron-like triggers and Claude's headless mode to operate day after day. For instance, the ClinicalTrials.gov connector and routines explicitly target "cron jobs" and scheduled runs【1】【5】, while the CureCancerMagic chain shows internal automation across a series of tasks【2】. In the HL7→FHIR example the entire migration pipeline could be invoked regularly via a GitHub trigger or schedule to process new data in bulk【6】.

**Relevance to Automated Oncology Trials:** These examples form a baseline for AI-managed oncology study workflows. A fully automated "physical AI" trial system might similarly use Claude Code routines to handle recurring tasks: polling EHR systems nightly for new lab results, summarizing physician notes, checking consent logs, or notifying staff of events. For example, a Claude agent could run each morning to triage patient-reported symptoms (analogous to the clinical trial finder), or to update consent status dashboards (like the daily compliance sweep in e-consent demos). The HL7-to-FHIR demo shows how one could automatically reconcile data formats with Claude Code in a loop. In short, by combining scheduled Claude Code routines with healthcare connectors and AI summarization, one can automate many daily trial operations. The 2026 projects demonstrate that domain experts can now build such agents themselves – rapidly prototyping and then deploying them as 24/7 processes【3】【5】.
