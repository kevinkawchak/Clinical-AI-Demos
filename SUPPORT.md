# Support

## Getting Help

This repository is maintained as an open-source resource for engineers and clinical trial operations teams building humanoid agents and LLM-driven Physical AI demonstrations for oncology clinical trials.

### Questions and Discussions

- **GitHub Issues**: Open an issue for bugs, prompt clarifications, or new demo proposals.
- **Discussions**: Use the GitHub Discussions tab (if enabled) for open-ended questions about humanoid platform selection, on-prem LLM strategy, or multi-site coordination.

### Before Opening an Issue

1. Confirm you have read the relevant prompt file in `demo-projects/`.
2. Check that you have access to the companion repository kevinkawchak/physical-ai-oncology-trials for the input file references.
3. Check existing [issues](https://github.com/kevinkawchak/Clinical-AI-Demos/issues) for similar reports.
4. Include the demo prompt number (01 through 10), your humanoid platform, your LLM backend, and the exact error or question.

## Regulatory and Clinical Questions

This repository provides **demonstration prompts and tooling references** for humanoid agents in oncology trials. It does not provide legal, medical, or regulatory advice. For compliance questions, consult your institution's:

- Regulatory affairs team (FDA, EMA submissions)
- IRB or Ethics committee (protocol review)
- Privacy officer (HIPAA, PHI handling)
- Quality management (IEC 62304, 21 CFR Part 11)
- Robotics safety officer (IEC 80601-2-77, ISO 10218, ISO 13482)
