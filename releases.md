Humanoid + LLM Oncology Trial Demo Prompts (v0.1.0)
v0.1.0 - Humanoid and LLM Demo Prompts for Physical AI Oncology Clinical Trials

## Summary

- Established the Clinical-AI-Demos repository as a companion to kevinkawchak/physical-ai-oncology-trials. The v0.1.0 release delivers 10 standalone Claude Code task brief prompts at `demo-projects/` that future Claude Code Opus 4.7 1M Max sessions read to author Physical AI oncology clinical trial demonstrations. No simulation files are produced by this PR; each future session reads the prompt and authors its demonstration in a separate downstream pass across seven sequential commits in a single pull request per prompt.
- Each of the 10 prompts has its own perspective spanning the new humanoid + LLM oncology trial industry: trial site operations director (01), pharmaceutical sponsor humanoid operations center (02), pharmacy and investigational medicinal product compounding (03), post-operative recovery nurse (04), biospecimen and pathology lab assistant (05), tele-surgical assistant (06), 24/7 adverse event response team (07), clinical research coordinator (08), radiation oncology technologist (09), and decentralized home care for decentralized clinical trials (10).
- Each prompt features a distinct humanoid platform: Boston Dynamics Atlas Electric (28 DOF, 1.5 m, 89 kg) for the site director and the radiation oncology patient handler; 5 Tesla Optimus Gen 3 humanoids (43 DOF, 1.73 m, 57 kg) at the sponsor operations center and one Optimus paired with Atlas at the LINAC vault; Figure 03 (32 DOF, 1.68 m, 60 kg) for CAR-T pharmacy compounding and (in Field Edition at 28 kg) for home DCT visits; Agility Digit V5 (16 DOF, 1.8 m, 80 kg) for the PACU recovery nurse role; Sanctuary Phoenix Gen 8 (24 DOF per hand, 1.7 m, 70 kg) for the CAP-accredited pathology lab; Apptronik Apollo (40 DOF, 1.73 m, 73 kg, 25 kg per-arm payload) for the rural tele-surgical assistant role; 1X Neo Beta (28 DOF, 1.65 m, 35 kg, soft tendon-driven joints with compliant skin) for the clinical research coordinator role; and 3 Unitree H2 humanoids (39 DOF, 1.8 m, 70 kg, outdoor-rated) for the 4-site 24/7 adverse event response team.
- Each prompt features a distinct large language model control loop: Claude Opus 4.7 1M context on-prem for the site director, the rural tele-surgical assistant, and the per-site AE response; Claude Sonnet 4.6 on-prem with Claude Opus 4.7 1M cloud failover (gated by Safe Harbor 45 CFR 164.514(b)) for the sponsor operations center; GPT-5.5 Thinking on-prem (Ollama-style local inference) for CAR-T compounding planning; Claude Haiku 4.5 default plus Claude Sonnet 4.6 escalation plus Ollama Llama 4 70B local for the PACU recovery nurse PHI inference path; Gemini 3 Pro cloud plus Ollama Qwen3 72B on-prem coordinated via Model Context Protocol (MCP) server for the pathology lab; a multi-model ensemble of Claude Opus 4.7 (medical reasoning), Gemini 3 Pro (multi-language consent in English, Spanish, Mandarin, Tagalog, Vietnamese), and GPT-5.5 Thinking (stipend payment guidance) for the CRC role; NVIDIA GR00T N1.6 humanoid foundation model plus NVIDIA Cosmos Reason 2 vision-language model plus Claude Opus 4.7 1M arbiter (running on a local NVIDIA DGX SuperPOD plus an on-prem Anthropic deployment) for the radiation oncology dual-humanoid pair; and Claude Haiku 4.5 edge-deployed on NVIDIA Orin AGX 64GB onboard the humanoid (with zero PHI egress and a federated learning aggregator on the sponsor side) for the home DCT cycle.
- Each prompt specifies exact directories and files from kevinkawchak/physical-ai-oncology-trials as inputs (collectively covering sponsor/, new-trial/, new-trial/national-24-7-trial/, new-trial/site/, patient-journey/, patients/paper/full-paper/sections/hr_9501_* through hr_9507_*, regulatory/Adaption-21-CFR-Part-312/, regulatory/Adaption-21-CFR-Part-50/, regulatory/adaption-ich-e6r3/, regulatory/ich-gcp/, regulatory-submit/, privacy/, federation/, digital-twins/, unification/usl/humanoids/, unification/surgical_robotics/, agentic-ai/examples-agentic-ai/, examples-new/, and competitions/instructions/). Each prompt also includes step-by-step downstream LLM processing instructions for the future Claude Code Opus 4.7 1M Max session: clone or update the companion repository, pre-chunk any file over 20K tokens, apply the three-layer chunking strategy from competitions/instructions/chunking_strategy.md, replace SVG diagrams with ASCII or Mermaid for high-frequency time series, author the future output tree under demo-projects/NN-name-output/ without modifying the companion repository, run the pre-commit CI checklist (ruff check, ruff format --check, yamllint -d relaxed .github/), and commit seven sequential commits on a branch named claude/demo-NN-name-shortid.
- Established complete repository scaffolding mirroring the kevinkawchak/physical-ai-oncology-trials companion repository pattern: ruff.toml with line-length 120 and target-version py310 plus appropriate per-file-ignores for demo-projects/ and national-repositories/; CI workflow at .github/workflows/ci.yml with the lint-and-format job on a Python 3.10/3.11/3.12 matrix (steps: actions/checkout@v4, actions/setup-python@v5, pip install ruff yamllint, ruff check . --output-format=github, ruff format --check ., yamllint -d relaxed .github/) that prevents the 3 failing checks (Cl / lint-and-format on 3.10, 3.11, 3.12) called out in the project brief; PR template with safety and compliance checklist plus CI checklist; bug report and feature request issue templates; CITATION.cff in Citation File Format 1.2.0; CONTRIBUTING.md with prompt authoring style guide; CODE_OF_CONDUCT.md adopting Contributor Covenant v2.1 with clinical research context; SECURITY.md covering humanoid agent safety, LLM control loop safety, and regulatory compliance; SUPPORT.md with humanoid vendor table and framework support channels.
- All 10 prompts use single dashes only, no em dashes, no double dashes, no triple dashes outside valid Markdown table separators. Black text only throughout. ASCII diagrams cap at 80 columns by 60 lines per competitions/instructions/ascii_diagram_guide.md of the companion repository. The release adds no Python, YAML, or other CI-checked files outside of markdown code blocks; the lint-and-format CI workflow (ruff format check, ruff check, yamllint) on Python 3.10, 3.11, and 3.12 remains green.

## Notes

The v0.1.0 release does not produce simulation files. The release delivers 10 self-contained Claude Code task brief prompts that future Claude Code Opus 4.7 1M Max sessions read alongside kevinkawchak/physical-ai-oncology-trials companion repository inputs to author the 10 humanoid + LLM oncology trial demonstrations across seven sequential commits in a single pull request per prompt. The 10 prompts inherit the three-layer chunking strategy from competitions/instructions/chunking_strategy.md of the companion repository, the SVG-to-ASCII replacement pattern from competitions/instructions/ascii_diagram_guide.md, the CI compliance pattern (ruff format, ruff check, yamllint on Python 3.10, 3.11, 3.12) proven in v3.6.0 through v3.9.1 of the companion repository, the seven-commit single-PR pattern from competitions/instructions/pr_workflow.md, the Gaussian skill rating model with mu_0 = 600 and sigma_0 = 200 from competitions/instructions/competition_protocol.md, and the three-category competitor model (prior versions, competitor configurations, hybrid teams). Each prompt defines its own iteration count (16 for most demos, 32 for the higher-resolution surgical demos), its own Parquet output schema, its own comparison weights across Quality / Time / Cost / Safety / Patient Experience plus demo-specific dimensions, and its own E-stop latency budget (5 ms surgical-grade for demos 03, 06, 07, 09; 10 ms benchtop-grade for demo 05; 50 ms site-grade for demo 01; 100 ms patient-interaction-grade for demos 02, 04, 08, 10). The future simulation passes for the 10 prompts will populate `demo-projects/01-humanoid-site-operations-director-output/` through `demo-projects/10-humanoid-decentralized-home-care-output/` with approximately 70 files each (700 total), approximately 80 MB committed Parquet plus CSV plus JSONL plus Markdown, and approximately 12 GB of Zenodo-archived raw telemetry across all 10 demos within the Zenodo free 50 GB tier. The CI lint-and-format job on Python 3.10, 3.11, 3.12 will continue to pass through all 10 future downstream PRs because the future sessions follow the ruff.toml per-file-ignores established in this v0.1.0 release.
