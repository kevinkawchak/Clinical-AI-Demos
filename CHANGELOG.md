### Notes
- All additions are Markdown only; no Python, YAML, or other CI-checked files are introduced outside of markdown code blocks. The lint-and-format CI workflow (ruff format check, ruff check, yamllint) on Python 3.10, 3.11, and 3.12 remains green.
- The v0.1.0 release does not produce simulation files. The release delivers 10 self-contained Claude Code task brief prompts that future Claude Code Opus 4.7 1M Max sessions read to author the simulations across seven sequential commits in a single pull request per prompt.
- Each prompt's downstream LLM processing instructions name specific input files from kevinkawchak/physical-ai-oncology-trials. The downstream sessions read those companion-repository files; they do not modify or commit anything to the companion repository.
- All 10 prompts use single dashes only, no em dashes, no double dashes, and no triple dashes outside valid Markdown table separators. Black text only throughout. ASCII diagrams cap at 80 columns by 60 lines per `competitions/instructions/ascii_diagram_guide.md` of the companion repository.
- Per-demo safety constraints (E-stop latency 5 ms to 100 ms, cumulative cross-arm force limits 5 N to 25 N, inter-humanoid minimum distances 8 mm to 1.5 m where applicable, PHI egress gates) are documented in each prompt's Notes section and aggregated in the demo-projects/README.md safety constraints matrix.
- Each prompt's comparison framework uses the three-category competitor model (prior versions, competitor humanoid configurations, hybrid humanoid + human teams) inherited from `competitions/instructions/competition_protocol.md` of the companion repository, with demo-specific weight allocations across Quality, Time, Cost, Safety, and Patient Experience plus demo-specific dimensions (Tele-Link Resilience for demo 06, Federated Aggregator Latency for demo 10, etc.).
- The seven-commit single-PR pattern documented in `competitions/instructions/pr_workflow.md` of the companion repository is the binding workflow for every downstream session executing one of these prompts.
- The v0.1.0 release places the new release notes in releases.md and the v0.1.0 block in CHANGELOG.md per the standard format from the kevinkawchak/physical-ai-oncology-trials companion repository.
